# REVIEW Mode

## Goal

Improve memory quality without erasing evidence or silently rewriting history.

## Workflow

1. Define project/repository scope and date range.
2. Retrieve active, contested, superseded, volatile, commitment, state, and recent episode memories.
3. Identify duplicate claims, orphan notes without source/scope, stale volatile facts, contradictions, outdated state snapshots, completed commitments, and repeated lessons.
4. Compare authority, evidence, scope, validity, and canonical status before proposing a merge or supersession.
5. Create corrections, superseding memories, relationships, procedures, rollups, or refreshed snapshots only through safe supported paths.
6. Require explicit human approval for deletion, bulk editing, provenance removal, or broad canonical rewrites.
7. Test retrieval after safe changes.

## Outcomes

- Same claim and scope, no new evidence: report duplicate; do not write.
- Same claim with stronger evidence: safely add evidence or link a supporting memory.
- Same claim in different scope: keep separate and clarify boundaries.
- Same topic but different event: keep distinct episodes.
- Changed authoritative outcome: create a correction or superseding memory.
- Unresolved conflict: mark contested and preserve both sources.
- Repeated verified lesson: consider promotion to a scoped procedure.
- Superseded or completed history: lower recall priority or archive safely; do not delete by default.

Return a compact report; do not dump note bodies.
