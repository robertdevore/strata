## AI Model Selector & Catalog

### Per-thread model selection
Users can now choose exactly which model powers a given chat thread. A dropdown next to the compose textarea shows "Auto" (default, uses the existing cheap/premium routing flow) or any configured model. Selecting a model persists to the thread in SQLite and survives app restarts.

The dropdown is fully custom-built — searchable with keyboard navigation (arrows, Enter, Escape, Tab), click-outside-to-close, themed scrollbar, min-width with ellipsis for long names, and animated caret-up/caret-down icons for open/closed state.

### User-configurable model catalog
Models available in the dropdown come from a new `aiModelCatalog` setting — a JSON object mapping provider IDs to comma-separated model names:

```json
{"openai":"gpt-5.3-codex, gpt-4o, gpt-4o-mini, o4-mini","deepseek-flash":"deepseek-v4-flash, deepseek-chat"}
```

Settings → AI → Model Catalog shows one text field per provider, pre-populated with IDE-common defaults. Add or remove models freely; clear a field to hide that provider from the chat dropdown entirely. Only configured providers appear in the selector.

The catalog builder on the backend merges user settings with preset defaults, ensuring the dropdown always has sensible content even before customization.

### Model label polish
Provider names in dropdown rows are kept short — "DeepSeek" instead of "DeepSeek Flash / DeepSeek Pro" — so model rows read cleanly at a glance.

### Fixes shipped
- **Catalog reloads on chat open** — previously loaded once at mount and never refreshed; now reloads each time the panel opens so settings changes take effect immediately.
- **Active state correctness** — eliminated a JavaScript falsy-string bug (`'' || threadModel`) that caused gpt-5.5 to appear permanently highlighted. Selection state now uses an `optimisticModel` local value as the single source of truth, decoupled from prop sync races.

## Hotkey Remapping

Three default hotkeys changed for better ergonomics:

| Action | Old | New |
|--------|-----|-----|
| Open Settings | *(none)* | `Cmd+Shift+S` |
| Toggle Star | `Cmd+Shift+S` | `Cmd+D` |
| Copy Rich Text | `Cmd+Shift+C` | `Cmd+C` |

`Cmd+C` for copy rich text is context-aware: only intercepts when the CodeMirror editor is focused. Everywhere else, native copy works normally.

The Settings → Hotkeys tab now includes all 17 bindings including the new "Open Settings" entry.

## AI Chat Improvements

### Wiki-link autocomplete in chat composer
Typing `[[` in the chat input now opens a note suggestion dropdown, identical to the editor's wiki-link behavior. Arrow keys navigate, Enter/Tab inserts `[[Note Title]]`, Escape dismisses, click selection works. Sources the same note list as the editor.

### User message bubble styling
User messages in chat are now right-aligned at 66.677% width with a subtle background tint. This visually separates your messages from the AI's responses, matching conventions from modern chat UIs.

### Chat persistence across preview toggle
Switching to Preview mode and back no longer resets the active chat thread. The thread and its messages persist through the toggle.

## Note Editing Reliability

### AI truthfulness guard
The AI runner now validates assistant responses against actual tool execution. If a model claims "Done — I updated the note" but no tool call actually modified a note, the response is replaced with an honest "No note content was changed" message. Combined with a system prompt update that explicitly forbids false success claims.

### Title-based note tools
Two new AI tools — `get_note_by_title` and `update_note_by_title` — let the assistant target notes by name (e.g., "TODO NOW") rather than requiring a raw UUID. Title resolution handles exact matches, substring matches, and ambiguity (multiple matches returns candidates). This is the CLI-equivalent path for note operations exposed through the tool API.

### Automatic retry for note edits
When a user requests a note edit and the model returns text without executing tools, the runner now automatically retries once with an instruction to use note tools before replying.

### Paste H1 title detection
Pasting markdown that starts with `# Title` now immediately flushes to the database — no 420ms debounce wait — so the sidebar and tab bar update instantly.

### List editing fix
CodeMirror's markdown extension was inserting blank lines between list items on Enter and paste. Fixed with:
- A `Prec.highest` keymap override that continues lists with single newlines
- A paste normalizer that collapses double newlines before list markers
